# Introdución

Os compoñentes personalizados creanse para poder facilitarnos a creación de interfaces xa que se utilizamos moito un compoñente con certas funcionalidades poderémolo reutilizalo sen ter que engadir ditas funcionalidades ao compoñente cada vez que o queiramos emplear.

Crear un compoñente dende cero é moi complicado asi que nós farémolo dende os compoñentes que netbeans e o seu GUI nos brindan, para así facilitarnos o traballo.

Estes compoñentes son os chamados Beans, falaremos a continuación deles e das súas características.

# Bean

Como dixemos anteriormente, un Bean é un compoñente reutilizable que pode ser manipulado visualmente por unha ferramente de linguaxe java.

As súas características máis salientábeis serían :

  • Persistencia : Debe poder gardarse o estado do compoñente.

  • Introspección : Debe permitir que o IDE reconoza os nomes das propiedades, o tipo de datos ...

  • Personalización : A apariencia e conducta do compoñente pode ser alterada.

  • Eventos : Debe avisar ao IDE das acción/sucesos que sofre dito compoñente.
